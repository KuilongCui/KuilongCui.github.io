本篇文章是对于《C++标准库体系结构与内核分析》的学习，参考资料为侯捷老师STL源码剖析，记录版本为G4.9

前驱知识

C++基本语法，C++模板

概览

Container通过Allocator取得数据存储空间，Algorithm通过Iterator存取Containter的内容，Functor可以协助Algorithm完成不同的策略变化，Adapter可以修饰或者套接Funator。

分配器

​ 可以看到allocate通过调用::operator new实现，而deallocator通过调用::operator delete实现，::operator new与::operator delete本质就是malloc与free

容器

list

list内部实现双向环状链表，sizeof取值为8->两个指针，不能随机存取，插入删除方便，内含sort()。

Vector

vector内含3个迭代器，start、finish（最后一个元素之后一个位置）和end_of_storage。当申请的内存不足时，如果原来的大小为0，则置为一个元素大小，或者变为原来的两倍。随机存取，插入删除需要移动元素，效率较低。vector::iterator为_Tp*外套一个iterator adapter，使其能够支持5种associated type。

arrary

arrary中没有构造函数和析构函数，arrary的[]运算没有进行边缘检测，而at()进行了边缘检测，其iterator为naive pointer。

forward_list

线性单向序列，不提供operator--()操作

deque

Deque内含有一个控制台map，由map_size个离散的buffer组成，每个buffer的大小为512Bytes，但是Deque需要对外表现为连续空间，通过迭代器实现此功能。在插入元素时，由于要移动元素，Deque还可以根据插入位置左右两边的元素个数，选择需要移动的方向。

queue

queue为容器适配器，不允许遍历，不提供迭代器，可以选择其他容器作为底层容器，但是底层容器必须含有对应的方法。

stack

stack与queue类似，不再赘述

rb_tree

按照正常的++ iterator即可获得排序状态，不应该使用iterator改变元素值，但是编程方面并未组织此事。
​rb_tree提供两种iterator操作：insert_unique(const Value &)和insert_equal(const Value&)。

Set/MultiSet

set/multiset更像是容器适配器，其value与key合一，value就是key。按照正常规则(++ite)，便可以获得排序状态。无法使用set/multiset的iterators改变元素值，其为const iterator。

Map/MultiMap

map/multimap排序依据为key，按照正常规则(++ite)，便可以获得排序状态。不能通过iterator改变key的值，但是可以改变data的值，内部的key type设为const。

Hashtable

Unordered Container

迭代器

迭代器连接算法与容器，算法做出提问，Iterator必须提供5种associated type以做出回答

Iterator Traits

Iterator traits必须能够分辨class iterators 和non-class iterator，两者行为方式不同

算法

Algorithms看不见Containers，对其一无所知；所以，它所需要的一切信息必须从Iterators钟获取。算法能不能判断出迭代器的类型对于效率具有重大影响，语法上也无法对于迭代器的类型做出强制要求，只能通过模板参数 的名称进行暗示。
