C++内存管理机制

先驱知识

操作系统，C++11

primitives

C++应用程序使用memory的途径：

分配 释放 类属 是否可重载
malloc() free(0) C函数 否
new delete C++表达式 否
::operator new() ::operator delete() C++函数 是
allocator::allocate() allocator::deallocator() C++标准库 可自由设计

new expression

delete expression

placement new

placement new允许我们将对象构建在allocated memory，没有对应的placement delete，或者说对应的operate detele就是对应的placemen delete。

::operator new/::operate delete

不要重载全局的operator new/delete，这影响过于深远；可以重载成员函数的new与delete，它们为隐式静态函数。

重载 new/delete

对于operator new函数或者operator new[]函数来说，它的返回类型必须是void*，第一个形参的类型必须是size_t且该形参不能含有默认实参。 对于operator delete函数或者operator delete[]函数来说，它们的返回类型必须是void，第一个形参的类型必须是void *。只有当new所调用的ctor抛出异常是，才会调用对应版本的operator delete，不写出对应版本的operator delete也可以，表示不处理该异常。

std::allocator

G4.9 allocator

G4.9的allocator只是::operator new和operator delete完成内存分配与释放，没有特殊设计。

G2.9 allocator

G2.9使用内存池的技术，每个相同大小的内存块由链表串联起来。当第二级内存分配器无能无力时，调用第一级内存分配器，不断尝试使用new_handle来获取内存。但是deallocate()并没有释放内存，设计上含有缺陷。

一个换肤工程，将分配单位由bytes改为元素个数

第二级分配器，两个模板参数由于与讨论内容关系不大，暂时忽略

malloc/free

VC6中的malloc/free在调试模式下使用了类似freelist的方法，同时使用bitmap表示每个list的使用情况；内存回收时，进行deferring，再回收一块内存，才会进行释放，此处不再赘述，**主要是不会**。

Loki allocator

loki allocator精简强悍，手段暴力，以array取代list,以index取代pointer，能够以很简单的方式判断chunk全回收，进而将内存还给操作系统，有deferring能力；但是他却使用了vector，What happends？

other issues

array allocator

array allocator从一个数组中分配内存，释放内存时，不会回收已分配的内存。

debug_allocator

可以认为是一个wrapper，在分配的内存前加一个额外空间，记录分配的大小，简单来说，没什么用。

bitmap_allocator

只为一次分配一个元素的需求服务
